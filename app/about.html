About Me (a few words)

I've returned to the Pacific Northwest, where I grew up, to work in downtown Seattle, for a San Francisco based software company, focusing on test automation (SDET). My work in Mexico has been some of the best in my life, but I felt the need to return home and be close to both my parents and my brother's growing family. I am excited about the change, somewhat sad to be out of Mexico and California, but look forward to what life brings and the beauty of the Puget Sound.

I orginally came to Chiapas from Berkeley, California in May of 2008 to build experimental software for a small research University, El Colegio de La Frontera Sur (ECOSUR). As part of my open source work with rules based gaming systems, I maintained a more public presence on the network by means of Twitter, Google Plus and other social systems. Please see the links in my Stream for deeper reading.

Open Source (mad science)

From 2008 until 2013, most of my work was public, open source code, hosted on GitHub. I keep three repositories there, one, "ecosur-sistemas-complejos" that contains the complex systems work that I built in collaboration with Max Pimm and Luis García Barrios; the other, "ecosur-infonomía", containing web projects built for the Infonomía (Infonomy) group. Finally, I keep a personal workspace under "awaterma" on GitHub with a variety of projects, this website, among them.

    Simple JavaScript application from the WSF API.
    Detail
    Ferrily.me runs on Node.js as a full-stack 'mobile-first' JavaScript application. It uses the super-fast LevelDB database as a cache for the JSON data that comprises each schedule. I'm currently using Hammer.js for managing touch events; although I'd like to reduce my dependency on external libraries in a future release.
